# ExaSense 統合エネルギー最適化プラットフォーム — システム構想

**作成日**: 2026-02-28
**ステータス**: 構想段階
**関連Issue**: #28, #29, #30, #31, #32, #33, #34

---

## 概要

ExaSenseを「工場屋根の太陽光シミュレーション」から「工場敷地全体のゼロカーボン達成支援プラットフォーム」へ拡張する構想。関西電力の法人向けソリューション（太陽光オンサイト・PPA・蓄電池・EV充電・空調熱源電化・ゼロカーボンコンサル）を1ツールで横断的に提案可能にする。

**一言で言うと**:
「ドローンで撮って → 3D化して → 太陽光/カーポート/蓄電池/EV/排熱を全部シミュレーションして → CO₂ゼロまでのロードマップをPDFで出す」プラットフォーム。

## レイヤー別詳細

### 1. Input Layer — データ取得

| データソース | 形式 | 用途 | 必須/任意 |
|-------------|------|------|-----------|
| ドローンRGB画像 | JPEG/PNG (10-50枚) | 3D再構築 | 必須 |
| サーマルカメラ画像 | TIFF/RAW | 排熱マッピング (#31) | 任意 |
| EPW気象ファイル | .epw | 実気象シミュレーション | 推奨 |
| 電力消費データ | CSV (30分値) | 自家消費率・ピークカット計算 | #29,#33で必要 |
| 電力料金体系 | YAML/JSON | 経済性評価 | 推奨 |

### 2. Spatial Analysis Layer — 空間解析

#### Roof Analyzer (実装済み: #13 RANSAC)
- メッシュからRANSAC平面検出で屋根面を分類
- 各面の面積・傾斜・方位を算出
- パネル設置可否の判定（傾斜角、面積閾値）

#### Ground/Parking Analyzer (新規: #28, #32)
- 地面平面検出 → 駐車場エリアのセグメンテーション
- 白線・区画の検出（点群の色情報 or VLMによるセマンティック理解）
- 車両動線・通路幅のパラメータ化
- **技術候補**: Open3D平面検出 + scikit-learn クラスタリング

#### Equipment Detector (新規: #32, #33)
- キュービクル（受変電設備）の位置特定
- パワーコンディショナー、分電盤の検出
- 蓄電池設置候補エリアの抽出（平坦・日陰・浸水リスク低）
- **技術候補**: 3D物体検出 or VLMによるセマンティックラベリング

#### Multi-Building Integrator (新規: #30)
- 複数フライト画像の統合アライメント
- 建物間のグローバル座標系での統合
- 建物間相互遮蔽のレイキャスティング
- **技術候補**: ICP (Iterative Closest Point) による点群位置合わせ

### 3. Simulation Layer — シミュレーション

#### Solar Simulation (実装済み)
- pvlib: 太陽位置、DNI/DHI/GHI、Perez拡散モデル
- trimesh: レイキャスティングによる面ごとの影行列
- セル温度補正 (SAPM/Faiman)
- Sky visibility matrix (Reinhart 580パッチ)

#### Carport Simulation (新規: #28)
- パラメトリックカーポート構造生成（柱位置・屋根角度・高さ）
- 駐車場点群 + カーポートメッシュの統合シーン
- 建物の影がカーポートに与える影響の時間帯別評価
- 屋根面 + カーポートの統合発電量算出
- **実装イメージ**: `src/simulation/carport.py`

#### Thermal Simulation (新規: #31)
- サーマル画像のキャリブレーション → RGB画像との位置合わせ
- VGGT点群に温度属性を付与
- 温度閾値ベースのホットスポット検出
- ヒートポンプ設置候補の空間的提案
- 排熱回収量 → 電力削減量の概算
- **実装イメージ**: `src/simulation/thermal.py`

#### Energy Flow Simulator (新規: 横断モジュール)
- 30分値の発電プロファイル × 消費パターン → 自家消費率
- 余剰電力 → 蓄電 or 売電の振り分け
- ピーク需要とのマッチング
- **実装イメージ**: `src/simulation/energy_flow.py`

### 4. Optimization Layer — 最適化

#### PPA Evaluator (新規: #29)
- 単年シミュレーション → 20年スパンへの拡張
- パネル劣化率 (年0.5-0.7%) の経年モデル
- 電力買取単価シナリオ（固定/変動/3パターン以上）
- NPV / IRR / LCOE の自動算出
- オンサイトPPA vs オフサイトPPA vs 自家消費の比較
- **実装イメージ**: `src/optimization/ppa_evaluator.py`
- **主要ライブラリ**: numpy-financial

#### Battery Optimizer (新規: #33)
- 3Dモデルから設置候補エリアを自動抽出
- 設置条件スコアリング（距離、荷重、温度、浸水リスク）
- 発電プロファイル + 消費パターン → 最適蓄電容量（線形計画法）
- ピークカット効果・自家消費率向上シミュレーション
- BCP観点：停電時バックアップ持続時間
- **実装イメージ**: `src/optimization/battery.py`
- **主要ライブラリ**: scipy.optimize or PuLP

#### EV Charger Optimizer (新規: #32)
- 駐車場3D点群からの区画・電気設備自動検出
- 配線距離コスト + 利便性スコアの多目的最適化
- 普通充電 / 急速充電の台数比最適化
- カーポート発電 → EV充電の直接供給シナリオ
- 充電需要予測（通勤パターン / 社用車稼働率）
- **実装イメージ**: `src/optimization/ev_charger.py`
- **主要ライブラリ**: scipy.optimize

#### Zero Carbon Roadmap Engine (新規: #34)
- 各モジュールのCO₂削減量を統一フォーマットで集約
- 投資額順 / CO₂削減効率順でのステップ最適化（ナップサック問題）
- 段階的ロードマップ自動生成 (Step 1→5)
- 2030年/2050年目標に対する達成度計算
- 累積投資額 × 累積CO₂削減量の時系列グラフ
- **実装イメージ**: `src/optimization/zero_carbon.py`
- **主要ライブラリ**: Plotly, numpy

### 5. Intelligence Layer — AI分析

#### VLM Analysis (既存 + 拡張)
- **現在**: 日照ヒートマップ画像 → 設置提案テキスト
- **拡張後**: 全シミュレーション結果 + ロードマップグラフを入力 → 総合コンサルレポート
- LoRAファインチューニングで関西電力ソリューション特化の文言を学習
- **モデル**: Qwen3.5-VL + Unsloth LoRA

### 6. Presentation Layer — 出力

#### Gradio WebUI (既存 + タブ追加)

| タブ | 内容 | ステータス |
|------|------|-----------|
| 3D再構築 | 点群・メッシュビューア | 既存 |
| 日照ヒートマップ | 年間日射量の3D表示 | 既存 |
| ソーラーカーポート | カーポート3D設計 + 発電量 | #28 新規 |
| 蓄電池・EV充電 | 配置計画 + 経済性 | #32,#33 新規 |
| PPA経済性 | 長期収益シミュレーション | #29 新規 |
| ゼロカーボンロードマップ | 段階的導入計画 + CO₂グラフ | #34 新規 |
| VLM総合レポート | AI自然言語レポート | 既存+拡張 |

#### PDF Report Generator (新規)
- 全タブの結果を統合した顧客提出用提案書
- 会社ロゴ・フォーマットのカスタマイズ
- **技術候補**: WeasyPrint or ReportLab

- **#32 EV充電最適配置**: 研究レベルの3D物体検出が必要
